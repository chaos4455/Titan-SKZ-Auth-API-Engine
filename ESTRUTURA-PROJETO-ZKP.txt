## ROTAS API
- POST /v6/auth/mint       — Legado (user/scope/entropy)
- POST /v6/zkp/identity    — Criar identidade (pubkey_pem)
- GET  /v6/zkp/challenge   — Obter nonce
- POST /v6/zkp/mint        — Mint ZKP (identity_id, nonce, signature)

## FLUXO STRESS TESTER
1. Conecta na API
2. Cria NUM_IDENTITIES via /v6/zkp/identity
3. Salva cada uma em u-data/entity_{i}/identity.json
4. Workers carregam identidades de u-data
5. Para cada request: GET challenge → assinar → POST mint
